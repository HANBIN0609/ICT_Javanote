상속의 기본문법 이해
 - 코드의 재활용을 위한 문법입니다. (X)
 - 연관된 일련의 클래스들에 대해 공통적인 규약을 정의할 수 있습니다. (O)
 - 
	class Man{
		. . .
	}
	class BusinessMan extends Man{
		. . .
	}
>>> 상속의 대상이 되는 클래스, 상위 클래스, 기초 클래스, 부모 클래스 : Man 클래스
>>> 상속을 하는 클래스, 하위 클래스, 유도 클래스, 자식 클래스 : BusinessMan 클래스

 - 부모거를 먼저 메모리에 올려놓고 자식거를 메모리에 올림.(메모리 그림 그릴 수 있도록.)
 - 상속하면 자동으로 super();를 컴파일러가 내부적으로 생성함.
	// 부모에 있는 변수 초기화 방법
	// 1. this로 접근(private 없을 경우)
	// 2. super();로 접근(private 있을 경우)
 - 자바는 다중 상속을 지원하지 않는다.(단일 상속만 지원함.)
   >>> 한 클래스에서 상속할 수 있는 최대 클래스의 수는 한 개.
 - 클래스 변수, 메소드는 프로그램 전체에서 단 하나만 있으므로 상속의 대상이 되지 않는다.
   >>> 그러나 접근 제한자가 허용하는 범위 내에서 접근은 가능함.
   >>> 제한자 protected(상속을 위한 access modifier)를 사용하면 package가 달라도 사용할 수 있다.
   >>> 객체지향 언어의 특징(OOP), 정보은닉
	- 정보를 노출(메모리에 write가 된다.)시키지 않는다.
	- 정보의 직접적인 접근(특히, 잘못된 접근)을 막기위해 문법사항을 추가해서 잘못된 접근이 생길경우 컴파일 에러를 낸다.
