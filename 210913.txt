참조변수의 매개변수(파라미터) 선언

null?
 - ex) BankAccount ref = new BankAccount( );
 - 주소값을 안넣고 있다
 - 즉, 참조 값이 없는 상태.(null은 참조형에서만 쓰인다.)
 - 위의 ref가 참조하는 인스턴스와의 관계를 끊음

생성자와 String 클래스의 소개
 - 코드상에서의 문자열 표현은 String 인스턴스의 생성으로 이어진다.
 - 문자열을 메소드의 인자로 전달할 수 있다.
 - 매개변수로 String형 참조변수를 선언하여 문자열을 인자로 전달받을 수 있다.
 - 생성자(함수) : new 옆에 오는 것
	- 생성자(함수) 내에 아무것도 안오는 것 : default 생성자
	- 생성자는 클래스의 이름과 동일해야 한다.
	- 개발자가 생성자를 하나라도 만들면 default 생성자를 만들지 않는다.
	- default 생성자의 생성조건 
		- 개발자가 생성자를 만들지 않는다.
		- 다른 개발자가 생성자를 만들지 않는다.
	- 생성자는 반환형을 받지않는다.
	- 생성자의 용도 : 객체 생성과 동시에 값을 넣어준다.(멤버변수 초기화)
*********** 모든 클래스의 인스턴스 생성은 생성자 호출을 동반한다.

클래스의 이름 규칙
 - 대문자로 시작한다.
 - 둘 이상의 단어가 묶여서 나타날 때는 새로 시작하는 단어를 대문자로 시작시킨다.(Camel Case모델)

메소드와 변수의 이름 규칙
 - 메소드 및 변수 이름의 첫 문자는 소문자로 시작한다.
 - 둘 이상의 단어가 묶여서 나타날 때는 새로 시작하는 단어를 대문자로 시작시킨다.

상수의 이름 규칙
 - 상수의 이름은 모든 문자를 대문자로 구성한다.
 - 둘 이상의 단어가 묶여서 나타날 때는 단어 사이를 언더바로 연결한다.
